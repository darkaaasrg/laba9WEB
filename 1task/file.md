**Сервіс автентифікація та користувачі**

1. Призначення

   Цей сервіс є ключовим бекенд-API, відповідальним за керування ідентифікацією та доступом. Його основні функції — обробка запитів на реєстрацію нових користувачів та автентифікацію (вхід) існуючих. Після успішного входу, сервіс генерує та повертає JWT-токен, який клієнт надалі використовує для доступу до захищених ресурсів інших API-сервісів.

2. Користувач сервісу

   Головним користувачем цього сервісу є Фронтенд-додаток ( React-додаток, що працює в браузері кінцевого користувача).

   Кінцевий користувач (людина) взаємодіє з формами на фронтенді. Фронтенд-додаток (на програмному рівні) робить прямі HTTP-запити до цього сервісу (наприклад, POST /api/register або POST /api/login) для отримання токена.
